# AI as search engine
2 years before, when I'm working with Peking University's AI4Math group in an annotation program.
I used a lot of an AI enhanced theorem search engine for Lean.
It's like one can use a very unclear pattern matching to find the exact theorem one want.
It's powerful and during this period I'm deeply convinced by the power of AI to replace the search engines nowadays.

Well, an interesting question is - what is a search engine?
Basically, a search engine can be viewed as a machine which you can give it a query and returns a list of ranked hyperlinks.
Hyperlinks leads you to the place where the resources lies in - a physical computer running software which task is to listen to the query and respond the content.

This is an old fashion library way to search for resources.
One has to know at least some key part of the resources they want.

Well, if we replace the physical computer running software with a language model.
Then the language model will somehow give us the "best guess" of what we want to ask.
We then look into the resources of those best guesses. We might not miss anything we are interesting in even if we only have a rough idea.

To me, this is what we can naturally do with AI to enhance the peformance of our search engine.
